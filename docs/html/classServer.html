Detailed Description

Главный класс сервера брокера сообщений.

Координирует работу всех компонентов системы:

  • Принимает входящие TCP-подключения от клиентов
  • Обрабатывает команды авторизации (REGISTER, LOGIN)
  • Обрабатывает команды обмена сообщениями (PUB, SUB, UNSUB, ACK)
  • Доставляет сообщения подписчикам в реальном времени
  • Восстанавливает пропущенные сообщения при переподключении (replay)
  • Повторно отправляет неподтвержденные сообщения (retry)
  • Выполняет периодическое компактирование хранилища
  • Обеспечивает корректное завершение работы (graceful shutdown)

Многопоточная архитектура:

  • acceptClients: поток приема входящих соединений
  • handleClient: по одному потоку на каждого клиента
  • consumerThread: извлечение сообщений из очереди и доставка подписчикам
  • retryThread: проверка pendingAck и повторная отправка (каждые 5 секунд)
  • compactThread: компактирование журнала (каждые 10 минут)
  • consoleThread: чтение команд администратора из stdin

Все разделяемые структуры данных защищены мьютексами для потокобезопасного доступа из нескольких потоков.

◆ retryThread()

void Server::retryThread ( )

Поток повторной отправки неподтвержденных сообщений.

Каждые 5 секунд проверяет pendingAck. Для сообщений, на которые не получен ACK более 5 секунд, выполняет повторную отправку. После 10 неудачных попыток сообщение удаляется принудительно.

◆ start()

void Server::start ( int  port)

Запускает сервер на указанном порту.

Последовательность запуска:

  1. Инициализация логгера
  2. Создание TCP-сокета
  3. Загрузка данных из файлов (пользователи, подписки, офсеты)
  4. Запуск фоновых потоков
  5. Переход в цикл приема подключений
