Detailed Description

Персистентное хранилище сообщений на диске.

Реализует append-only журнал с индексом для быстрого доступа. Все сообщения дописываются в конец файла data.log, что обеспечивает высокую скорость записи за счет линейного доступа к диску.

Формат записи в data.log: [MessageHeader][topic байты][body байты]

Индекс index.idx хранит отображение ID сообщения на смещение в файле, что позволяет читать любое сообщение за константное время без полного сканирования журнала.

Удаление сообщений мягкое: запись помечается флагом deleted в индексе, но физически остается в журнале до вызова compact().
