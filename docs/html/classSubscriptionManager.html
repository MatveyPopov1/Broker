Detailed Description

Менеджер подписок потребителей на топики.

Управляет двумя структурами данных:

  • subs: активные подписки (топик -> список потребителей)
  • knownTopics: все когда-либо существовавшие топики

Топики создаются автоматически при первой подписке или публикации. При отписке последнего потребителя топик удаляется из subs (active), но сохраняется в knownTopics для отображения в LIST_TOPICS. Это позволяет новым пользователям видеть полный список существующих топиков и подписываться на них осознанно.

Все публичные методы потокобезопасны (защищены std::mutex). Данные персистентно хранятся в файлах:

  • subscriptions.dat: активные подписки
  • topics.dat: все известные топики
